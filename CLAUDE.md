## 1. PROJECT OVERVIEW

**Momentum Capital** is a public-facing SaaS dashboard for a quantitative momentum rotation trading strategy. It shows live ETF + crypto theme picks, momentum rankings, backtest performance, and pricing — all built to eventually gate content behind Stripe subscriptions.

### What It Does
- Displays **current theme picks** (1–2 ETFs or direct crypto) generated by `v2_picks_generator.py`
- Shows **19 theme momentum rankings** (ranked by 16–26 week price return, EMA 10/100 daily filter)
- Presents **19-year backtest results** (2006–2025: 47.72% Balanced / 61.34% Growth CAGR vs SPY 10.80%)
- Explains the strategy (How It Works, FAQ, Resources)
- Lists subscription tiers (Free / Pro $49/mo / Premium $99/mo) — **not yet wired to Stripe**

### Tech Stack

## 3. ARCHITECTURE & DATA FLOW

### Full Pipeline
```
GitHub Actions (Mon–Fri 5pm ET)
    ↓  runs scripts/v2_picks_generator.py
    ↓  reads universe_config.json for constituent stocks (falls back to hardcoded)
    ↓  fetches prices for 19 themes + ~150 constituent stocks from EODHD
    ↓  ranks themes by momentum, applies EMA 10/100 filter
    ↓
data/picks_raw.json  ←  updated via GitHub Contents API (curl PUT) — NOT git commit/push
    ↓                    commit message includes [skip ci] [vercel skip] to skip Vercel redeploy
    ↓                    Vercel "Ignored Build Step" skips builds from github-actions[bot]
GET /api/picks (app/api/picks/route.ts)
    ↓  production: fetches GitHub raw URL (repo must be public)
    ↓  local dev: reads data/picks_raw.json, falls back to ../picks_raw.json
    ↓  normalizes field names, adds last_generated from file mtime
    ↓
DashboardPage (app/dashboard/page.tsx)
    ↓  fetches /api/picks every 60s via useCallback + setInterval
    ↓
PicksTable / ThemeRankings / PortfolioAllocation / RebalanceTimer

GitHub Actions (1st of month)
    ↓  runs scripts/rebuild_universe.py
    ↓  fetches top 15 holdings by weight per ETF from EODHD components API
    ↓
data/universe_config.json  ←  committed to repo, read by v2_picks_generator.py
```

### API Route: `/api/picks`
- **Production**: fetches from GitHub raw URL (`https://raw.githubusercontent.com/dylboy001/-momentum-dashboard/main/data/picks_raw.json`) — Vercel cannot read committed files at runtime
- **Local dev**: reads `data/picks_raw.json` then falls back to `../picks_raw.json`
- Cache: `next: { revalidate: 300 }` (5 min) — data updates show within 5 min of scanner run
- Configurable via `PICKS_JSON_PATH` env variable
- Returns 503 if no file found; 500 on parse error
- **Repo must stay PUBLIC** — raw.githubusercontent.com returns 404 for private repos without auth

### API Route: `/api/equity-curve`
- Accepts `?mode=balanced` (default) or `?mode=growth`
- Reads `data/equity_curve_data.json` (balanced) or `data/equity_curve_growth.json` (growth)
- Returns `{ data: [{date, strategy, spy}], final: {strategy, spy} }`
- These are static files — regenerate only if backtest numbers change

### picks_raw.json Shape
```json
{
  "date": "2026-02-27",
  "rebalance_next": "2026-05-28",
  "theme_rankings": [["SLV", 61.82], ["URA", 28.81], ...],
  "top_themes": ["SLV"],
  "picks": [
    {
      "ticker": "AG",
      "theme": "SLV",
      "price": 12.34,
      "rs_score": 15.2,
      "volatility": 42.1,
      "weight_pct": 33.3
    }
  ],
  "spy_momentum": -1.2,
  "universe_full_data": {
    "SLV": [{ "ticker": "AG", "price": ..., "rs_score": ..., "selected": true }]
  }
}
```

### PageHeader Component
Used by all inner pages (not homepage/dashboard). Renders animated title + optional subtitle — **no dot grid, no tag, no violet orb**:
```tsx

  sub="Optional subtitle text."
/>
```
The `tag` prop exists but is NOT rendered — it was removed to avoid a visual split between the header and page content. The page-level ambient blobs provide all the glow.

### Animation System
- **Homepage**: **No entry animations.** All content is immediately visible on load. `framer-motion` is only used for `whileHover`/`whileTap` on CTA buttons (interaction only, not entry). `AnimatedCounter` in StatsSection counts up when scrolled into view — the number itself is always visible, just animates the count. DO NOT add `initial={{ opacity: 0 }}` or `whileInView` entry animations to homepage sections — they caused a visible 2-second blank delay that was unacceptable.
- **Performance page**: `framer-motion` `whileInView` scroll-triggered reveals on all sections (safe because these sections are always below the fold)
- **Background**: `AnimatedWavesBg` (canvas-based wavy grid, used in Hero + CTA)
- **CSS-only**: keyframes in globals.css (`pulse-glow`, `mesh-float-a/b`)
- **Shared easing**: `[0.16, 1, 0.3, 1]` (expo out) used across all framer-motion transitions
- **Pulsing dots**: `animate-ping` on a violet or emerald dot = "live" indicator

### Data Fetching Pattern (client-side)
```tsx
const [data, setData] = useState<DataType | null>(null)
const [loading, setLoading] = useState(true)
const [error, setError] = useState<string | null>(null)

const fetchData = useCallback(async () => {
  try {
    const res = await fetch('/api/picks', { cache: 'no-store' })
    const json = await res.json()
    if (!res.ok) {
      setError(json.detail ?? json.error ?? 'Failed to load')
    } else {
      setData(json)
      setError(null)
    }
  } catch {
    setError('Network error')
  } finally {
    setLoading(false)
  }
}, [])

useEffect(() => {
  fetchData()
  const id = setInterval(fetchData, REFRESH_INTERVAL)
  return () => clearInterval(id)  // Always cleanup!
}, [fetchData])
```

## 6. CRITICAL RULES

### DO ✅
- Use `bg-[#080808]` (NOT `bg-black` or `bg-zinc-950`) for page backgrounds
- Use `font-mono tabular-nums` for all numeric data (prices, percentages, scores)
- Add `shrink-0` to icons inside flex containers to prevent squishing
- Use `transition-colors` on all interactive elements
- Always cleanup intervals: `return () => clearInterval(id)`
- Use `{ cache: 'no-store' }` on all fetch calls (data changes frequently)
- Add `last:border-0` on border-separated lists to avoid double borders
- Use `text-zinc-600` (not `-700` or `-800`) for the faintest readable text
- Test loading state with `<Skeleton className="... bg-zinc-800" />`
- Use `rounded-2xl` for major cards, `rounded-xl` for smaller elements, `rounded-lg` for inputs/badges
- Use `overflow-hidden` on basic card wrappers (not needed on GlassCard — it has its own)
- Export named functions (not default arrow functions) for components: `export function NavBar()`
- Use `<GlassCard>` for all dashboard and performance page sections
- Center all ambient blobs with `left-1/2 -translate-x-1/2`

### DON'T ❌
- **Don't use `localStorage` or `sessionStorage`** — not supported in this deployment context
- **Don't add `<form>` tags for data-display pages** — forms belong only in contact/auth flows
- **Don't use `bg-black`** — always `bg-[#080808]`
- **Don't hardcode color hex values in className** — use Tailwind unless you need `bg-[#specific]`
- **Don't use React Router** — this is Next.js App Router, use `Link` from `next/link` and `usePathname`
- **Don't add** `console.log` statements to production code
- **Don't use `any` type** — TypeScript strict mode is on
- **Don't put THEME_NAMES in individual component files** — import from `@/lib/theme-data`
- **Don't create new pages without `NavBar`** — every page uses `<NavBar />`
- **Don't use `motion.form`** — framer-motion is for visual reveals only
- **Don't nest recharts `<Legend>` inside charts** — use custom legend components (see EquityCurveChart)
- **Don't assume spy_momentum is always present** — it can be `null`
- **Don't use `display: none` for mobile hiding** — use Tailwind responsive prefixes (`md:hidden`, `hidden md:flex`)
- **Don't place ambient blobs off-center** — `left-1/3` or `left-[5%]` creates a visible purple patch on the left
- **Don't use `bg-[#0c0c0c]` on homepage panels** — ALL homepage panels use `bg-[#080808]`, the slight difference is visible and looks wrong
- **Don't add dot grids, violet orbs, or overflow-hidden to PageHeader** — it was removed to prevent a visual split
- **Don't change the dashboard layout without explicit instruction** — it went through many iterations to reach its current state

### Update Picks Data
Run the **v2 picks generator** from `~/`:
```bash
python v2_picks_generator.py           # Balanced mode (top 2 themes, 50/50)
python v2_picks_generator.py --growth  # Growth mode (top 1 theme, 100%)
```
- Fetches live prices for all 19 themes (17 ETFs + BTC + ETH) from EODHD
- Ranks by 20-week momentum, applies EMA 60/250 trend filter
- **Step 4 also fetches ~170 constituent stocks** across 17 themes (0.15s delay each) — total runtime ~5–7 min
- Writes to `~/picks_raw.json` including populated `universe_full_data` (price, RS score, volatility per stock)
- The Next.js API auto-reads the new file (no restart needed)
- The dashboard refreshes every 60 seconds automatically
- **Do NOT set `universe_full_data: {}` again** — it was hardcoded empty before and was intentionally fixed

> The old `Universe_generator_dynamic.py` used the v1 stock-picking strategy and is superseded by `v2_picks_generator.py`.

## 9. STRATEGY CONTEXT

Use this when generating marketing copy, FAQ answers, or educational content.

### Core Strategy — V2 (ETF + Crypto Rotation, Updated)
**Momentum rotation** — systematic rebalancing into the strongest market themes.
No individual stock picking. Pure ETF + direct crypto rotation.

**How it works:**
1. Rank all 19 themes (17 ETFs + BTC + ETH) by momentum (16w Balanced / 26w Growth)
2. Apply EMA 10/100 daily filter: theme must have Price > EMA(10d) > EMA(100d)
3. Select top 1–2 qualifying themes
4. Balanced mode: equal-weight top 2 (50/50) | Growth mode: 100% top 1
5. Rebalance weekly (Balanced, every 7d) or monthly (Growth, every 30d)

**Key formulas:**
```
Momentum Score (Balanced) = (price_today / price_16w_ago) - 1
Momentum Score (Growth)   = (price_today / price_26w_ago) - 1
EMA filter                = Price > EMA(10 days) > EMA(100 days)
```

**Backtest (2006–2025, 19 themes, no stocks):**
| Metric | Balanced | Growth | SPY |
|---|---|---|---|
| CAGR | **47.72%** | **61.34%** | 10.80% |
| Sharpe Ratio | **1.163** | **1.065** | — |
| Max Drawdown | **-46.4%** | **-75.9%** | -55.2% |
| $10k → | **$21.5M** | **$121.9M** | $75k |

**Market regime performance (Balanced mode):**
- 2008 Financial Crisis: Strategy **-4.6%** vs SPY -40.4% (outperformed)
- 2022 Bear Market: Strategy **+11.6%** vs SPY -16.6% (outperformed)
- 2023 Recovery: Strategy **-2.8%** vs SPY +24.3% (underperformed — AI mega-cap rally)
- 2024: Strategy **+44.4%** vs SPY +28.8% (outperformed)

**The 19 themes:**
```
US Equity Sectors (11):
  XLK  → Technology       XLF  → Financials       XLV  → Healthcare
  XLE  → Energy           XLY  → Consumer Cyclical XLP  → Consumer Defensive
  XLI  → Industrials      XLB  → Materials         XLU  → Utilities
  XLRE → Real Estate      XLC  → Comm. Services

Commodities & Crypto (8):
  GLD  → Gold     SLV  → Silver    CPER → Copper    URA  → Uranium
  ICLN → Clean Energy              BITO → Crypto Futures
  BTC  → Bitcoin (direct)          ETH  → Ethereum (direct)
```

**Key insight — why v2 beats v1:**
Stock selection is net negative for CAGR (9.74% with stocks vs 47.72% without).
Crypto allocation is the primary alpha driver — but only when the EMA filter allows it.
The EMA 10/100 daily filter blocked crypto during 2018–19 and 2022–23 bear markets.
Previous EMA 60/250 was on weekly bars (≈ 14mo/4.8yr EMAs) — essentially no filter. Daily EMAs are the correct interpretation.

### Animation Rules

**What to animate:**
- ✅ `transform` (translate, scale, rotate) — GPU-accelerated
- ✅ `opacity` — GPU-accelerated
- ⚠️ `transition-colors` — fine for hover state color changes
- ❌ `transition-all` — triggers layout recalc on every property; **avoid for new code**

**Replace `transition-all` with specific properties:**
```tsx
// ❌ Avoid
className="transition-all duration-200"

// ✅ Prefer — be explicit about what's changing
className="transition-colors duration-200"
className="transition-transform duration-200"
className="transition-[colors,shadow] duration-200"
```

**Standard easing:** `cubic-bezier(0.16, 1, 0.3, 1)` — already the project standard.
In framer-motion: `const E = [0.16, 1, 0.3, 1] as const`

**Duration guidelines:**
| Type | Duration | Examples |
|---|---|---|
| Micro-interaction | `duration-200` | Color change, border, icon swap |
| Hover lift / shadow | `duration-200` to `duration-300` | Card translate, glow |
| Page reveals | 600–800ms | Performance page scroll-triggered reveals |

**Loading states:** Always `<Skeleton>`, not full-page spinners.
Exception: Small `animate-spin` on a single icon (e.g., `<RefreshCw className="animate-spin" />`) is fine for inline status indicators.

### Homepage Mobile (`app/page.tsx`)

**Decision**: All 5 sections shown on all screen sizes. No sticky stacking, no hidden sections.

`app/page.tsx` is a simple flat stack — no wrappers, no sticky, no hidden:

```tsx
export default function HomePage() {
  return (
    <div className="bg-[#080808] text-white">
      <NavBar />
      <HeroSection />
      <ProductClaritySection />
      <StatsSection />
      <HowItWorksSection />
      <CTASection />
    </div>
  )
}
```

**Why no sticky stacking**: Removing it required all section components to have zero entry animations (no `initial={{ opacity: 0 }}`). Sections that are in the initial viewport with `whileInView` cause a brief blank flash as IntersectionObserver fires asynchronously after mount. Removing sticky stacking means sections may be in initial viewport — so all homepage entry animations were stripped.

### Hero (`components/homepage/HeroSection.tsx`)

All content is plain HTML — immediately visible on load. No framer-motion entry animations. Only the bouncing scroll arrow uses `motion.div` (decoration, not content). `AnimatedWavesBg` handles the canvas background. Section min-height: `min-h-[85vh] sm:min-h-screen`

### Rankings Page Mobile (`app/rankings/page.tsx`)

Flex layout on mobile, CSS grid on `md+`. This avoids horizontal overflow from fixed-width grid columns:

```tsx
{/* Header + rows use the same pattern */}
<div className="flex items-center gap-2 md:grid md:grid-cols-[56px_1fr_130px_150px_160px_160px] md:gap-4 px-4 md:px-6 ...">
  <div className="w-8 md:w-auto ... shrink-0">#</div>
  <div className="flex-1 min-w-0 ...">Sector</div>
  {/* Category, vs SPY columns: hidden md:block */}
  <div className="hidden md:block ...">Category</div>
  <div className="hidden md:block ...">vs SPY</div>
  {/* Momentum + Status: shrink-0 to stay on same row */}
  <div className="... shrink-0">Mom.</div>
  <div className="... shrink-0">Status</div>
</div>
```

Sector name truncation: `truncate` on the name `<div>` inside `flex-1 min-w-0`.

---

### No Horizontal Scroll Rule

- Never use fixed-width grid columns (`grid-cols-[Npx ...]`) at base breakpoint — use `md:grid-cols-[...]`
- Always pair `flex-1` with `min-w-0` when the item should truncate
- Add `shrink-0` to icons and short fixed-width items (rank numbers, status labels, momentum values)
- Use `overflow-x-auto` only as a last resort for tables that truly can't be simplified
- Prefer hiding columns on mobile over making the table scrollable
